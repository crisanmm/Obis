Abstract

Obis is an application that let's you view and access statistics about obesity in the United States. Data is collected by telephone surveys, the data is provided by Behavioral Risk Factor Surveillance System (BRFSS).

Introduction

Obesity in the United States is a major health issue resulting in numerous diseases, specifically increased risk of certain types of cancer, coronary artery disease, type 2 diabetes, stroke, as well as significant increases in early mortality and economic costs.

"Obesity" is defined as a Body Mass Index (BMI) of 30 and over, and the measure of "Adult Obesity" from the BRFSS encapsulates the self-reported prevalence of obesity among adults for the civilian non-institutionalized population 18 years and over

In order to make it easy for everyone to see and use the data BRFSS gathered, we offer:

  • An API that can be accesed while having an account
  • Different charts to display the situation in USA
  • Comparisions between different states
  • Means to download the charts in 3 different formats

Our goal is to facilitate an easy interaction between the given data and users, using self explanatory, visual charts.

Project Requirements

Make a WEB application to visualise BRFSS data with the following specifications:

  • Create a REST API
  • Comparisions between states
  • 3 different charts to display data
  • Facilitate chart export as CSV, WebP and SVG

Technologies Used

  • HTML

    • HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content.

  • CSS

    • Cascading Style Sheets (CSS) is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

  • JavaScript

    • JavaScript is a prototype-based, multi-paradigm, single-threaded, dynamic language, supporting object-oriented, imperative, and declarative styles. We choose JS because this project required a programming language capable of making dynamic and interactive web pages by implementing custom client-side scripts.

  • PHP

    • We choose PHP because is a robust programming language and it has an extensive list of core functionality which includes cross-platform adaptability, multi-browser compatibility, secure web apps and so on. Compared to other web languages, PHP is an efficient language which supports all major web servers to deliver high performance consistently

  • MySQL

    • It came with the XAMPP pack, and it is exceptionally quick, regardless of the underlying platform. It features self-management capabilities like auto restart, space expansion and automatic configuration changes for ease of management.

  • Apache

    • Like the database, the Apache server came along with XAMPP. With the tool ngrok, we were able to redirect localhost to a public adress. Apache proved to be reliable and secure

  • AJAX

    • AJAX allows web pages to be updated asynchronously by exchanging small amounts of data with the server behind the scenes. For us to be able to update the charts without refreshing the page we choose to use AJAX.

  • REST

    • One of the main features of our website is the API we offer. In order to achieve a complete set of capabilities we created a RESTful API utilising CRUD functionality.

  • CSV

    • CSV files are plain-text files, making them easier for the website developer to create. The data we got from BRSFF was in a csv format, making it simple to upload it to our database.

  • JSON

    • The JSON format is often used for serializing and transmitting structured data over a network connection. It is used primarily to transmit data between a server and web application, serving as an alternative to XML.

  • JWT

    • We choose to generate a token for every user in order to acces our API. This keeps the database secure and avoids bloated/spam like requests.(bots)

Code Management System

For this project we decided we are going to use github. Git is the most commonly used version control system. Git tracks the changes you make to files, so you have a record of what has been done, and you can revert to specific versions should you ever need to. Git also makes collaboration easier, allowing changes by multiple people to all be merged into one source. This proved to be crucial because different changes were added at the same time, as we worked in parallel. We were able to track commit changes and discover where an issue was created, making it easier to solve, without changing large chunks of code.
